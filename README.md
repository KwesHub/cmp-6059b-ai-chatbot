# CMP-6059B AI Chatbot — Group Project

Advanced Artificial Intelligence coursework (CW2).
A conversational chatbot for a train operating company built with
Python, spaCy, experta, Keras, SQLite, and Streamlit.

## What it does

- **Task 1** — Finds the cheapest available train ticket for any UK journey via the National Rail OJP SOAP API. Handles single and return tickets, date/time selection, and station disambiguation across ~2,600 UK stations.
- **Task 2** — Predicts the arrival delay for a train on the Weymouth → London Waterloo route using a Keras MLP model trained on historical performance data.
- **Knowledge Base** — Answers common train-related questions. New Q&A rules can be added live via the "add a rule" dialogue.

## Project Structure

```

├── app.py                  # Streamlit UI — main entry point

├── requirements.txt
│
├── nlp/
│   ├── intent.py           # spaCy intent classifier

│
├── engine/

│   └── knowledge_base.py   # KB Q&A pairs and fallback rules (SQLite-backed)
│
├── task1/
│   └── ticket_api.py       # National Rail OJP SOAP API + fare parser
│
├── task2/
│   ├── clean_data.py       # Merge and clean raw CSV files

│   ├── keras_model.py      # Keras MLP training and export
│   └── predict.py          # Model loader used by the chatbot at runtime
│
├── data/                   # NOT in GitHub — see Data files section below
└── models/                 # NOT in GitHub — generated by training scripts
```

## Setup: do this once

```bash
# 1. Clone
git clone https://github.com/KwesHub/cmp-6059b-ai-chatbot.git
cd cmp-6059b-ai-chatbot

# 2. Install dependencies
pip install -r requirements.txt

# 3. spaCy language model
python -m spacy download en_core_web_sm

# 4. Set up environment variables (National Rail API credentials)
# Create a .env file in the project root:
# NATIONAL_RAIL_USERNAME=your_username
# NATIONAL_RAIL_PASSWORD=your_password

# 5. Initialise the database (creates chatbot.db and seeds station codes)
python database.py

# 6. Run the chatbot
streamlit run app.py
```

## ⚠️ Demo day — do this before presenting

Model files and data CSVs are gitignored. On the demo machine you must run:

```bash
# 1. Put the CSV files from Blackboard into data/

# 2. Preprocess the data
python task2/clean_data.py
python task2/preprocess.py

# 3. Train the models (generates files in models/)
python task2/train_models.py
python task2/keras_model.py

# 4. Delete any old database so stations re-seed cleanly
rm -f chatbot.db

# 5. Run
streamlit run app.py
```

## Data files

The raw CSV files (`2022-2025_service_details.csv`) are **not** in this repo.
Download from Blackboard and place in the `data/` folder before running the preprocessing scripts.
